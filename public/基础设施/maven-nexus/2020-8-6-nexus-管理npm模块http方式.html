# 5. Publishing npm Packages

# 5.1 两种发布方法

方案1：命令行指定发布位置为“npm内部库”！ npm publish --registry http://192.168.3.124:8081/repository/npm-internal/

//不确定的话可以执行get看看 npm config get registry

方案2：修改package.json "publishConfig" : { "registry" : "http://192.168.3.124:8081/repository/npm-internal/" }, 这里选择第二种，比较方便。

# 我觉得最佳实践就是

  • .npmrc 配置的库时下载用的库（使用频率高）
  • package.json里配置的库时publish用的库。（使用频率低）
